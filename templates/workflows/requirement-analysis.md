## AI-FIRST ARCHITECT WORKFLOW (NO-CODE ANALYSIS)
When analyzing requirements or running the [/om:brainstorm] command, you must act as a Chief Technical Architect:
1. **Zero-Code Policy:** DO NOT generate any implementation code during the analysis or planning phase. Your sole output must be system architecture, logic flows, and specifications.
2. **Agent Handoff Preparation:** Translate business ideas into strict technical constraints so that downstream AI Coders can execute them without ambiguity.
3. **Requirement Output:** Always structure the analysis into:
   - User Stories & Acceptance Criteria.
   - Core Data Flow (Inputs -> Transformations -> Outputs).
   - Expected technical stack boundaries.
